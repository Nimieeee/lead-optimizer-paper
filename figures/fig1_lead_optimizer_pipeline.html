A vision-language agentic pipeline for lead optimization

Twelve deterministic stages map a lead structure and its ligand-interaction diagram to a ranked set of synthetically-accessible analogs. Three substrate categories, upstream open-source primitives, Benchside-implemented logic, and defence-in-depth gates, are visually distinguished. Stage indices match backend/app/services/lead_optimizer/orchestrator.py.

in

Inputs

Lead SMILES (required)  ·  Ligand-interaction diagram (PNG, optional)  ·  Project context (free-text, therapeutic indication, target class, constraints, e.g. "CNS DYRK1A inhibitor, prioritize BBB")  ·  Visual hints  ·  Analog budget

1

Pre-scan

RDKit · MurckoScaffold · 90+ SMARTS substructure library

Computes Bemis–Murcko scaffold, detects functional groups, assigns 2D coordinates and per-instance position hints (phenyl_left vs phenyl_right).

RDKit 2024.03 SMARTS lib
2

Vision Agent

Vision-language classification · structured JSON output

Classifies functional groups in the LID image into RESTRICTED (binding-contact) and TARGET (editable), with per-group residue and interaction-type lists. Outputs a strict-schema record that downstream stages validate before use.

two-category per-instance multi-residue
2′

Chemistry validator

Allowlist tables: H-bond donor / acceptor / π-stack / hydrophobic

Defence-in-depth. Drops physically impossible classifications (e.g. methyl as H-bond donor, methoxy as donor). Drops groups absent from RDKit's detected set. Runs before any downstream stage trusts the vision call.

drop-on-fail
3

SMARTS builder (per-instance + scaffold anchor)

MolFragmentToSmarts (atom-anchored) · Murcko scaffold appended to restricted-SMARTS set

Constructs atom-specific SMARTS per labelled instance. The Bemis–Murcko scaffold SMARTS is appended to the restricted set, so any analog that breaks a scaffold atom is rejected later by the pharmacophore gate (stage 7). The actual soft Murcko gate fires on each candidate analog at validation and permutation time.

atom-anchored scaffold-locked
4

Lead ADMET profile

admet_ai (Chemprop v2 GNN) · 54-endpoint prediction batch

Single forward pass through the upstream ADMET predictor on the lead molecule. Output drives the optimization agent's liability-prioritisation prompt and the per-endpoint thresholds used in the screen at stage 10.

admet_ai 1.x
5

Project-context analysis

Structured-output language model · indication-aware ADMET weighting

Reads the user's project-context text (e.g. "CNS kinase, BBB-priority") and emits a typed ContextAnalysis record: per-endpoint priority weights, hard-stop thresholds (e.g. hERG > 0.5 hard-rejects), and a primary-optimization goal. This is where the indication shapes the rest of the pipeline.

user-context driven indication-aware
6

Optimization agent

Structured-output language model · ranked SAR strategies (JSON-strict)

Given the ADMET profile, project-context priorities, and labelled TARGET sites, proposes a ranked list of SAR strategies (lower LogP, halogen-to-cyano, ring isostere, etc.) mapped to SMIRKS-library categories. Strategies are emitted under a strict JSON schema.

structured proposals
7–8

Strategy validation + permutation (soft Murcko gate fires here)

479-entry SMIRKS library · RDKit reaction enumeration · enforce_pharmacophore (called twice)

Each proposed strategy is validated by enumerating one product and checking pharmacophore preservation (call #1). The permutation engine then expands the validated strategies across labelled TARGET sites; for every analog, enforce_pharmacophore rejects products that broke a RESTRICTED atom or the appended Murcko scaffold (call #2). Site × strategy × per-strategy cap bounds the combinatorial explosion.

SMIRKS-driven pharmacophore-locked ring-topology
9–10

Pre-filter + ADMET screen

Lipinski · PAINS · Brenk · admet_ai re-prediction

Hard structural filters drop invalid valencies, PAINS, Brenk alerts, and ADMET-failing analogs (directional scoring: hERG / Skin / CYP risk vs HIA / BBB benefit).

PAINS Brenk directional
11

Synth-accessibility

SYBA (Voršilák 2020, Bayesian) · Ertl SAScore (1–10) fallback

SYBA signed score (~−50 easy to ~+50 hard) is the platform-ratified synth-accessibility metric. Sigmoid-centered penalty feeds into ranking.

SYBA SAScore
12

Ranking and report

Weighted scalar sum × penalty multiplier · WeasyPrint PDF / HTML / CSV

Single-score "Pareto-style penalty rank", sorted total_score over docking, ADMET, SYBA, and structural axes. Top-K written to PDF/HTML/CSV with per-analog SMILES, modifications, SYBA verdict, ADMET profile, and structure depiction.

SYBA-anchored multi-format
out

Outputs

Ranked analog list  ·  per-analog ADMET + SYBA dossier  ·  PDF / HTML / CSV reports  ·  email delivery

1 Per-instance labelling. Two phenyl rings on the same molecule would collapse to a single generic SMARTS (c1ccccc1) without disambiguation. The pipeline derives left/right/top/bottom position hints from RDKit-generated 2D coordinates and emits atom-anchored SMARTS via MolFragmentToSmarts, so each instance carries a distinct restricted/editable verdict.

2 Soft Murcko gate, where it actually lives. The Bemis–Murcko scaffold SMARTS is appended to the restricted-SMARTS set at stage 3. The gate itself is implemented by enforce_pharmacophore(), called twice, once at strategy validation (does this SMIRKS, applied to one example, preserve the pharmacophore?) and once for every analog during permutation. Predicates checked: (a) restricted atoms unchanged, (b) ring-system topology preserved, (c) aromaticity preserved. Allows topologically equivalent edits (methyl→ethyl on a scaffold carbon); rejects ring destruction.

3 Chemistry validator coverage. Allowlist tables for H-bond donor, H-bond acceptor, π-stack, hydrophobic, salt bridge, and cation-π. Each entry maps a functional group family to its allowed interaction types. Vision-model output that violates the chemistry tables is dropped before SMARTS construction (stage 3), so a methyl group cannot enter the pipeline as a hydrogen-bond donor.

4 SMIRKS library scope. 479 curated SMIRKS transformations across 22 categories (aromatic-ring swaps, amine modifications, carbonyl modifications, CNS penetration, nitrogen-heterocycle swaps, polarity adjustments, halogen substitutions, steric shielding, metabolic stability, etc.) in the version inventoried for this paper. All entries carry a validated=True flag, but validation is curator-asserted, not gated by an automated RDKit-reaction-fires-on-target CI test, known limitation, fix planned before the supplementary release.

5 Ranking metric. The total_score is a weighted scalar sum across docking, ADMET liabilities, SYBA penalty, and structural alerts, multiplied by hard-filter penalties. The output is single-sorted; the field label "Pareto" is API-convention rather than a Pareto front. Honest framing in the paper: weighted scalar penalty rank with SYBA as the synth-accessibility anchor.
